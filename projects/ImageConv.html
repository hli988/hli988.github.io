Multithreading Image Convolution

  • Various image effects can be created by performing a convolution.

  • Convolution is a mathematical operation which describes a rule of how to combine two functions or pieces of information to form a third function. The feature map (or input data) and the kernel are combined to form a transformed feature map. The convolution algorithm is often interpreted as a filter, where the kernel filters the feature map for certain information. A kernel, for example, might filter for edges and discard other information.

  • Figure 1: Convolving an image with an edge detector kernel.

1. Task Description

  • Given an input image, an output image is constructed by computing each output pixel value as a weighted summation of the corresponding input pixel and the pixels forming its immediate neighbourhood in the input image.
  • Use multithreading to speed up the execution of convolution. Provide a graph of the relative speedup of multithreaded versions over the single-threaded version for 1–8 threads.

iii. Performance Analysis

  • From the test result, it can be observed that one thread takes a longer time than the others. As the number of threads increases to 2, the average execution has an obvious decline. For number of threads from 2 to 8, their performance is almost the same. Since a program is divided into sequential part and parallel part. The speedup is limited. In this test, the best performance is 2247.2 milliseconds while 5 Threads are used.
